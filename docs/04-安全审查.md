# FlowSync 安全审查报告

> 审查日期：2026-07-20
> 审查范围：后端服务、认证、数据库、外部 API、前端
> 审查人：TRAE 安全审查员

---

## 一、问题清单（按优先级排序）

### 🔴 严重（上线前必须修）

#### 问题 1：JWT Secret 默认值兜底，生产环境可能用弱密钥

- **入口**：`server/src/middleware/auth.js:4`
- **模块**：auth 中间件
- **行为**：`process.env.JWT_SECRET || 'your-secret-key-change-in-production'` — 若环境变量未设，会用硬编码默认值。攻击者知道默认值后可伪造任意用户的 JWT Token，完全绕过认证
- **影响**：任意用户冒充，全权限访问
- **修复方案**：生产环境启动时检查 `JWT_SECRET` 是否设置且长度≥32，否则直接 `throw` 拒绝启动

#### 问题 2：API Key 通过请求头明文注入 process.env，存在并发污染

- **入口**：`server/src/app.js:156-171`
- **模块**：全局中间件
- **行为**：用户自定义 API Key 通过 `X-API-Keys` 请求头写入 `process.env`，多用户并发时 A 的 key 会被 B 覆盖，导致：
  - 串号：A 用户的请求用到 B 的 API Key
  - Key 泄露：A 的 key 可能被 B 的请求日志记录
- **影响**：API Key 串号、泄露
- **修复方案**：改为 `req.apiKeys` 上下文传递，service 层从 `req` 取 key，不写全局

#### 问题 3：demo 接口限流过宽，LLM 成本失控

- **入口**：`server/src/routes/demo.js:13-29`
- **模块**：demo 路由
- **行为**：每分钟 5 次/每小时 30 次的限流太宽松，且内存限流在 Railway 多实例下失效；恶意用户可消耗大量智谱 API 额度，直接造成经济损失
- **影响**：API 额度被刷爆，服务不可用
- **修复方案**：收紧到每分钟 3 次/每小时 10 次 + 生产环境禁用 demo 接口或加 Cloudflare Turnstile

---

### 🟠 中等（上线后一周内修）

#### 问题 4：SQLite 单文件存储，Railway 容器重启数据全丢

- **入口**：`server/src/db.js:5`
- **模块**：数据库
- **行为**：数据库存在本地文件 `flowsync.db`，Railway 容器重启后数据清空，所有用户数据丢失
- **影响**：用户数据全失，产品不可用
- **修复方案**：Railway 挂载 Persistent Volume（/data 目录），DB_PATH 指向持久卷

#### 问题 5：错误信息直接暴露给客户端

- **入口**：`server/src/app.js:189-196` 全局错误中间件
- **模块**：错误处理
- **行为**：`err.message` 原样返回，可能暴露 SQL 错误、文件路径、数据库结构等内部信息，便于攻击者侦察
- **影响**：信息泄露
- **修复方案**：生产环境（`NODE_ENV=production`）只返回通用错误信息"服务器内部错误"，详细错误只记日志

#### 问题 6：注册接口无任何限流，可被批量注册

- **入口**：`server/src/routes/auth.js:40`
- **模块**：auth 路由
- **行为**：`/api/auth/register` 无 rate limit，可被脚本批量注册垃圾账号，污染数据库
- **影响**：数据库被垃圾账号填满
- **修复方案**：同 IP 每小时最多 3 次注册

---

### 🟡 低（可留作已知限制）

#### 问题 7：用户名/邮箱格式未校验

- **入口**：`server/src/routes/auth.js:44-50`
- **模块**：auth 路由
- **行为**：只校验非空和密码长度≥6，没校验邮箱格式、用户名合法字符
- **修复方案**：加正则校验

#### 问题 8：CORS 设为 `*` 在生产环境过宽

- **入口**：`server/src/app.js:32`
- **行为**：环境变量默认 `*`，任意网站可调用 API
- **修复方案**：生产环境改为具体域名白名单

#### 问题 9：无 HTTPS 强制跳转

- **行为**：Railway 默认 HTTPS，但 HTTP 请求也能访问
- **修复方案**：加 HSTS 头和 HTTP→HTTPS 跳转中间件

#### 问题 10：日志可能泄露敏感信息

- **入口**：`server/src/services/aiEngine.js:14`
- **行为**：`console.warn('[AIEngine] LLM_API_KEY 未设置，process.env.LLM_API_KEY =', process.env.LLM_API_KEY)` 可能打印 key 值
- **修复方案**：只打印是否设置，不打印值

---

## 二、修复计划

### 本轮修复范围

- **严重问题**（1-3）：全修
- **中等问题**（4-6）：全修
- **低等问题**（7-10）：暂不修，留作已知限制，上线后再迭代

### 存储方案

- 采用 Railway Persistent Volume 挂载 `/data` 目录，SQLite 数据库文件指向 `/data/flowsync.db`
- 本地开发保持原路径，通过 `DATABASE_PATH` 环境变量切换

---

## 三、审查结论

当前代码在比赛 demo 阶段可用，但作为真实产品对外提供服务前，严重和中等问题必须修复。修复后可进入 QA 自检和部署阶段。
