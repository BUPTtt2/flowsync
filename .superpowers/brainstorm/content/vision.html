FlowSync 大改版：从「单任务管理」到「全生活编排」

我对你需求的理解 — 看完告诉我哪里理解错了，哪里漏了

现在的 FlowSync

  • 用户手动创建任务，AI 帮你拆解
  • 一次聚焦一件事，做完下一件
  • 按能量排序当天任务
  • 语音输入 → 结构化解析
  • 单维度：今天的待办

改版后的 FlowSync

  • 用户说一句话 → 系统自动生成整个项目模板
  • 多项目并行编排，自动排期
  • 按 DDL + 准备时间 + 难度 + 能量综合排序
  • 外部 API 接入（高德地图等）自动补全信息
  • 多维度：面试 / 比赛 / 旅行 / 日常 全部在一个系统里

你提到的三个场景

🎯

面试 + 比赛并行

同时 3 个面试 + 2 个比赛，每个都有 DDL、准备时间、难度。系统自动生成每个面试/比赛的准备模块，按优先级排期，冲突时提醒你取舍。

✈️

旅行规划

说一句「下周末去成都 3 天」，系统自动生成：交通（飞机/高铁查询）、酒店、景点、美食、每日行程。出行前自动提醒，接入高德 API 推荐通勤方式。

⚡

更多场景（系统扩展）

考试复习、项目交付、搬家准备、活动策划... 系统不预设场景，而是根据用户输入动态生成结构化模板。AI 识别意图 → 自动生成模块。

新架构：三层联动

输入层 用户说一句话：「下周三有腾讯面试，还要准备华为比赛，月底想去成都玩」
↓
AI 核心层 1. 识别出 3 个独立项目
2. 每个项目自动生成结构化模板（面试有「技术复习 / 项目复盘 / 模拟面试 / 自我介绍」等模块）
3. 根据日期自动排期，标记冲突
4. 接入外部 API 补全信息（高德查交通、天气等）
↓
输出层 用户看到的是：一个完整的时间线，每个项目按模块展开，可以快速确认/修改/跳过，不需要自己从零创建任何节点

关键能力变化

能力现在改版后
任务创建用户手动建系统自动生成模板
任务拆解AI 拆成子任务AI 生成完整项目结构（含检查点、准备阶段、DDL）

我的理解对吗？

核心变化是：从「用户建任务 → AI 辅助」变成「用户说一句话 → AI 生成整个项目结构 → 用户确认/调整」。 而且这不是一个新模块，而是整个产品的交互逻辑都要改。
